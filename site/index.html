Works everywhere

One-time setup for each tool. Copy the config, paste, done.

Run this in your terminal. That's it.

terminal
# Basic claude mcp add --transport stdio credsmcp -- npx credsmcp # With password in environment claude mcp add --transport stdio \ --env CREDSMCP_PASSWORD=your-password \ credsmcp -- npx credsmcp # Global (all projects) claude mcp add --transport stdio --scope user credsmcp -- npx credsmcp

Add to your Claude Desktop config file:

macOS: ~/Library/Application Support/Claude/claude_desktop_config.json
Windows: %APPDATA%\Claude\claude_desktop_config.json

claude_desktop_config.json
{ "mcpServers": { "credsmcp": { "command": "npx", "args": ["credsmcp"], "env": { "CREDSMCP_PASSWORD": "your-password" } } } }

Restart Claude Desktop after saving.

Add to .cursor/mcp.json in your project root:

.cursor/mcp.json
{ "mcpServers": { "credsmcp": { "command": "npx", "args": ["credsmcp"], "env": { "CREDSMCP_PASSWORD": "your-password" } } } }

Or use Settings > Tools & Integrations > New MCP Server.

Add to your Windsurf MCP config:

macOS: ~/.codeium/windsurf/mcp_config.json
Windows: %USERPROFILE%\.codeium\windsurf\mcp_config.json

mcp_config.json
{ "mcpServers": { "credsmcp": { "command": "npx", "args": ["credsmcp"], "env": { "CREDSMCP_PASSWORD": "${CREDSMCP_PASSWORD}" } } } }

Windsurf supports ${VAR} interpolation. Set the env var in your shell profile.

Add via the Codex CLI:

terminal
$ codex mcp add credsmcp -- npx credsmcp

Set CREDSMCP_PASSWORD in your shell profile or ~/.codex/config.toml.

Create .continue/mcpServers/credsmcp.yaml in your project:

credsmcp.yaml
name: credsmcp command: npx args: - credsmcp env: CREDSMCP_PASSWORD: your-password

MCP tools are available in Continue's Agent mode (not chat mode).

Add any provider

One file. One interface. Automatically available to all tools.

Every provider implements the CloudProvider interface:

src/providers/base.ts
interface CloudProvider { name: string; // "AWS" slug: string; // "aws" description: string; requiredFields: FieldDefinition[]; optionalFields: FieldDefinition[]; verify(creds): Promise<VerifyResult>; toEnvVars(creds): Record<string, string>; }

Example: adding AWS as a provider:

src/providers/aws.ts
import type { CloudProvider } from "./base.js"; export const aws: CloudProvider = { name: "AWS", slug: "aws", description: "Amazon Web Services", requiredFields: [ { key: "access_key_id", label: "Access Key ID", secret: true }, { key: "secret_access_key", label: "Secret Access Key", secret: true }, ], optionalFields: [ { key: "region", label: "Default Region", secret: false }, ], async verify(creds) { // Call AWS STS GetCallerIdentity return { ok: true, message: "AWS verified" }; }, toEnvVars(creds) { return { AWS_ACCESS_KEY_ID: creds.access_key_id, AWS_SECRET_ACCESS_KEY: creds.secret_access_key, }; }, };

Then add it to src/providers/registry.ts — import and push to the providers array. Rebuild, done.

Security first

Your credentials never leave your machine unencrypted.

Encryption

AES-256-GCM with 12-byte random IV and 16-byte authentication tag. Tamper-proof — any modification invalidates the ciphertext.

Key Derivation

scrypt with N=217 (131,072), r=8, p=1. Memory-hard — requires 128MB RAM per derivation. Resistant to GPU and ASIC attacks.

Storage

~/.credsmcp/vault.json with 0600 permissions. Atomic writes via temp file + rename — no corruption on crash or power loss.

Redaction

API keys shown as 8a06...7384 by default. Full reveal requires explicit reveal: true parameter.

Vault file format — what's actually on disk:

~/.credsmcp/vault.json
{ "version": 1, "kdf": "scrypt", "kdfParams": { "salt": "a3f8...hex...9b21", "N": 131072, "r": 8, "p": 1 }, "payload": { "iv": "b7c4...hex...2e91", "tag": "f1a2...hex...8d3c", "ciphertext": "9e2f...hex...4a7b" } } // That's it. Just hex. No plaintext ever touches disk.

Master password resolution order:

1 CREDSMCP_PASSWORD environment variable
2 OS keychain (macOS Keychain / Windows Credential Manager / Linux libsecret)
3 Error with setup instructions (never prompts interactively)
